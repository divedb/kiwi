146// Returns a closure that, when run on any sequence that allows blocking calls,
147// will kick off a potentially asynchronous operation to delete `path`, whose
148// behavior is similar to `DeleteFile()` and `DeletePathRecursively()`
149// respectively.
150//
151// In contrast to `DeleteFile()` and `DeletePathRecursively()`, the thread pool
152// may be used in case retries are needed. On Windows, in particular, retries
153// will be attempted for some time to allow other programs (e.g., anti-virus
154// scanners or malware) to close any open handles to `path` or its contents. If
155// `reply_callback` is not null, it will be posted to the caller's sequence with
156// true if `path` was fully deleted or false otherwise.
157//
158// WARNING: It is NOT safe to use `path` until `reply_callback` is run, as the
159// retry task may still be actively trying to delete it.
160BASE_EXPORT Function<void()> GetDeleteFileCallback(
161 const FilePath& path, Function<void(bool)> reply_callback = {});

164 const FilePath& path, Function<void(bool)> reply_callback = {});

208// Copies a single file. Use CopyDirectory() to copy directories.
209// This function fails if either path contains traversal components ('..').
210// This function also fails if |to_path| is a directory.
211//
212// On POSIX, if |to_path| is a symlink, CopyFile() will follow the symlink. This

214//
215// If |to_path| already exists and is a regular file, it will be overwritten,
216// though its permissions will stay the same.
217//
218// If |to_path| does not exist, it will be created. The new file's permissions
219// varies per platform:
220//
221// - This function keeps the metadata on Windows. The read only bit is not kept.
222// - On Mac and iOS, |to_path| retains |from_path|'s permissions, except user
223// read/write permissions are always set.
224// - On Linux and Android, |to_path| has user read/write permissions only. i.e.
225// Always 0600.
226// - On ChromeOS, |to_path| has user read/write permissions and group/others
227// read permissions. i.e. Always 0644.
